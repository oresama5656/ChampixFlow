# PLAN: ChampixFlow スタンドアロンWebアプリ移行計画

## 🎯 目的
現在の「Node.js + SQLite3（クライアント・サーバー構成）」から、リフィル処方箋アプリ（`refill_manager`）と同様の**「完全クライアントサイド（サーバーレス）構成」**へ移行する。
これにより、**「アプリのプログラム本体はConoHaサーバーのHPに置き、保存データ（個人情報）だけは各PCのローカルフォルダにJSONファイルとして記録する」**という、究極のプライバシー保護と利便性の両立を実現する。

---

## 🛠️ 技術的アプローチ方針

1. **React UIの維持（デザインの保護）**
   - 現在の美しいUI（ダークテーマ、Ganttカレンダー、タブ構成）は**完全維持**する。
   - `refill_manager` のように「単一のHTMLファイル」に全てを書き直すのではなく、現在の「React + Vite」で構築し、ビルドして出来上がった静的ファイルセット（`dist` フォルダの中身）をConoHaサーバーにアップロードする方式を取る。

2. **通信（API）層の置き換え**
   - 現在の `fetch('/api/...')` を全て廃止する。
   - 代わりに、`refill_manager` で使用されている **File System Access API**（`window.showDirectoryPicker()`）を用いたローカルファイル（例: `champix_data.json`）の直接読み書き処理に置き換える。

3. **保存処理の変更**
   - バックエンドのSQLiteが行っていた「ステータスの自動更新」「スケジュールの自動計算」「ガントチャートバーの生成ロジック」を、すべてフロントエンド（React / JavaScript）側に移植する。
