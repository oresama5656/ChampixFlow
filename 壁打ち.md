# チャンピックス服用管理アプリ「ChampixFlow」要件定義書

## 1. プロジェクト概要

チャンピックス（バレニクリン）の特殊な服用スケジュール（漸増投与）に伴う「在庫管理の煩雑さ」と「患者の服用ミス・離脱リスク」を解消するための、薬剤師支援管理システム。

---

## 2. ターゲットユーザー

- **薬剤師**: 各患者の12週間の服用プランを管理し、将来の必要量を予測したい。
- **患者**: 服用の手引きとなるスタンプカードを受け取り、モチベーションを維持したい。

---

## 3. 確定した用法・用量

| 期間 | 用法 |
|------|------|
| Day 1-3（導入期） | 0.5mg 1日1回（食後） |
| Day 4-7（増量期） | 0.5mg 1日2回（朝夕食後） |
| Day 8-84（維持期） | 1.0mg 1日2回（朝夕食後） |

- 合計投与期間: **12週間（84日間）**

### スターターキット（初回交付）
- **内容**: 0.5mg錠×11錠 ＋ 1.0mg錠×3錠（2週間分セット）
- 初回は必ずこのセット品を交付。2回目以降は交付日数を手入力。

---

## 4. 主要機能

### ① 12週間スケジュール自動計算エンジン
- **入力**: 服用開始日・初回交付日数
- **出力**: 84日分の日付と用量を格納したタイムラインデータ

### ② ガントチャート型スケジュール管理画面
- **2ペイン構成**: 左=患者リスト、右=カレンダー
- **カレンダーナビ**: 前月/次月ボタンで1ヶ月単位のページ切り替え
- **個別バーの2段階表示**:
  - **実線バー（Solid）**: 今回交付済みの期間（患者名を表示）
  - **透明バー（Dashed/枠線）**: 次回交付予定期間（前回と同じ日数分を自動配置）
- **在庫集計行（ヒートマップ）**:
  - カレンダー最下部に「全患者の1.0mg錠 必要量の合算」をヒートマップで表示
  - 患者の個別バーは短くしつつ、将来の予約状況は集計行で把握できる
- **患者リストのON/OFFトグル**:
  - 離脱懸念・完了患者をトグルでカレンダー上の表示/非表示切り替え
  - 非表示にしても在庫集計の計算には含め続ける（在庫漏れ防止）

### ③ 患者向けスタンプカード印刷（A5横向き）
- **上部**: 患者名、服用開始日、禁煙達成予定日（大きく印字）
- **中央**: 12マスのスタンプエリア（第1週〜第12週）
  - 各マスに服用量アイコン（💊）と日付を自動印字
  - 完了済みの週には「済」マークを自動印字
- **下部**: 薬局名・電話番号・「吸いたくなった時のコツ」固定文言
- 週替わりメッセージを現在の週に応じて自動挿入

### ④ 離脱検知（デッドマンズスイッチ）
- 次回来局予定日を過ぎても「来局済み」に更新されない患者を「離脱懸念」として強調表示
- 離脱確定時、予約分の在庫集計から除外する

### ⑤ 完遂セレブレーション
- 第12週完了時に「禁煙達成証明書」を自動生成・印刷

---

## 5. 確定した技術仕様

| 項目 | 内容 |
|------|------|
| プラットフォーム | ローカルPCで動作するWebアプリ（デザイン重視） |
| 技術スタック | Node.js + Express + SQLite3 / Vite + React + Tailwind CSS |
| データ保存 | ローカルPC内SQLite（外部クラウド不使用） |
| 印刷仕様 | A4用紙横に置いてA5スタンプカード2枚取り（またはA5縦横） |
| バックアップ | CSV/JSON書き出し・読み込み機能対応 |
| 在庫管理 | 「理論上の必要量（予約数）」のみ管理。実在庫との同期なし |

## 6. タブ・ナビゲーション構成（refill_managerに準拠）

| タブ | 内容 |
|------|------|
| 📋 一覧・カレンダー | メイン画面。左=患者リスト、右=ガントチャートカレンダー |
| 📝 新規登録 | 患者名・服用開始日・初回交付日数の入力フォーム。登録と同時に印刷ボタンも配置 |
| ⚙️ オプション | 薬局情報（名称・住所・TEL）の設定。localStorage保存 |
| 📜 履歴 | 完了・離脱確定済みの患者アーカイブ |
| 📥 CSV書き出し | データのCSV/JSONエクスポート（バックアップ用） |

---

## 7. 確定した運用ロジック

- **来局遅延時の計算**: 12週間の治療枠は固定。次回の透明バーは**実際の来局日（交付日）を起点**に再計算。
- **開始日・処方実績の修正**: 修正時、未来の予約データを**全自動で再計算**。
- **処方日数の柔軟対応**: 毎回交付時に「何日分」を手入力できる。入力日数に応じて実線バーが伸び、そこから次の透明バーが自動配置される。
- **在庫データの扱い**: 実在庫数との同期は行わない。患者ごとの理論上の必要量だけを表示・管理する。

---

## 8. 患者ステータス定義

| ステータス | 条件 |
|-----------|------|
| 準備中 | 服用開始日前 |
| 進行中 | 12週間以内のアクティブ患者 |
| 離脱懸念 | 次回来局予定日を過ぎた患者 |
| 完了（卒業） | 12週間の処方を完遂した患者 |

---

## 9. 開発フェーズ（スプリント）

1. **第1フェーズ（基盤）**: 患者登録、12週間スケジュール算出、一覧表示
2. **第2フェーズ（UI）**: ガントチャート表示（実線・透明バー・集計行）
3. **第3フェーズ（帳票）**: スタンプカードのPDF生成・印刷
4. **第4フェーズ（運用）**: 離脱検知、完了証明書、週替わりメッセージ
